Critical Sync Question

When changes happen in Kajabi (outside your admin portal), how should we detect and handle them?

Example Scenarios:
  • Member cancels their subscription directly through Kajabi checkout page
  • Member updates their email in Kajabi
  • Subscription payment fails (Stripe/Kajabi auto-pauses)
  • Member manually resumes from their Kajabi account
  • Admin processes refund directly in Kajabi

How Should We Detect External Changes?

A

Kajabi Webhooks (Real-time)

How it works: Kajabi sends HTTP POST to your API when events occur (subscription.paused, member.updated, etc.)

Pros:
  • Real-time sync (seconds)
  • Event-driven architecture
  • No polling overhead
  • Can trigger Slack actions immediately
Cons:
  • Need to verify Kajabi has webhook support
  • Must handle webhook validation/security
  • Need retry logic for failed webhooks
  • Requires publicly accessible endpoint
B

Polling Kajabi API (Scheduled)

How it works: Cron job runs hourly/daily to fetch changed records from Kajabi (e.g., GET /members?updated_since=...)

Pros:
  • Simple, predictable
  • No special infrastructure needed
  • Can use Vercel Cron (free)
  • Easier to debug/test
Cons:
  • Delayed sync (minutes to hours)
  • API rate limit concerns
  • May miss rapid changes
  • Unnecessary API calls if no changes
C

Hybrid: Webhooks + Polling Backup

How it works: Use webhooks for real-time sync, but also poll periodically to catch missed events or webhooks failures

Pros:
  • Best of both worlds
  • Resilient to webhook failures
  • Fast for normal cases
  • Catches edge cases
Cons:
  • More complex to implement
  • Need idempotency everywhere
  • Higher maintenance
  • Two sync paths to debug
D

Admin Portal Only (No External Sync)

How it works: Disable direct member access to Kajabi settings. All changes MUST go through admin portal.

Pros:
  • Simplest architecture
  • Admin portal is guaranteed source of truth
  • No sync conflicts
  • Easiest to implement
Cons:
  • Members can't self-service
  • Payment failures still happen externally
  • Stripe webhooks bypass Kajabi
  • May frustrate members

Note: We can start simple and add complexity later. The architecture should support evolving from option D → B → A/C over time.
