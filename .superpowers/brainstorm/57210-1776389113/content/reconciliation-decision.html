Full Daily Reconciliation

Check ALL active members once per day for sync drift

What this means:

  • Daily cron job fetches all members from Kajabi
  • Compare Kajabi state vs Admin Portal state
  • Detect and fix any drift (missed webhooks)
  • Simple to implement, optimize later if needed

When to revisit:

  • If daily reconciliation takes > 5 minutes to run
  • If hitting Kajabi API rate limits
  • If you have > 1000 active members

Until then: Keep it simple!
